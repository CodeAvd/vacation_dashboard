# Skill And Agent Map

## Conductor

- Use `agent-delivery-orchestrator` when a task spans design, engineering, QA, ops, or business work and needs sequencing plus handoffs.

## Dashboard And Frontend

- Start with `frontend-art-direction-lead` when the page needs a visual thesis, breakpoint hierarchy, or anti-generic UI guardrails.
- Use `react-performance-surgeon` when the page is slow, over-hydrated, chunk-heavy, or structurally monolithic.
- Use `browser-ux-qa-auditor` after implementation for viewport, accessibility, console, and interaction acceptance.
- Pull in `web-design-guidelines` for review-style audits against a stricter UI rule set.
- Pull in `vercel-react-best-practices` when React or Next.js performance patterns need a second opinion.

## GTM And Content

- Start with `business-dev-systems-operator` for partnerships, outbound systems, funnel mechanics, and reusable operating packets.
- Add `agent-content-localization-lead` when RU or EN messaging, support flows, or terminology consistency matter.
- Add `humanizer` for public-facing copy that feels too synthetic after drafting.

## Blockchain And Crypto

- Start with `blockchain-product-architect` for custody, payments, on-chain integrations, reliability, and control design.
- Add `mcp-toolchain-architect` if the delivery depends on new tooling, monitoring, or ops integrations.
- Add `agent-qa-release-manager` before launch or risky rollout.

## MCP And Tooling

- Use `mcp-toolchain-architect` for stack decisions, setup runbooks, smoke tests, and ROI-based tool additions.
- Keep `agent-mcp-ops-engineer` as the deeper setup and troubleshooting specialist when an MCP is already selected.

## Default Handoff Order

1. `agent-delivery-orchestrator` if scope spans multiple lanes.
2. Domain lead agent for the main decision.
3. Implementation or optimization agent if code changes are required.
4. `browser-ux-qa-auditor` or `agent-qa-release-manager` for final acceptance.
